# Press v0 — Negotiation Phase, Token Commitments, and Détente Re-anchor

**Status:** Draft, pending dbyrne review.
**Date:** 2026-04-27.
**Authors:** David Derner, Claude (foedus session).

## 1. Motivation

Empirical observation from 2026-04-27 onboarding: under the v1 ruleset,
random-vs-random play **always ends in détente at turn 5** across every
seed tried (1, 7, 11, 42, 99). The détente trigger fires on passive
inaction, not on negotiated peace, so the strongest equilibrium under
any minimally-cautious play is "do nothing, win by timeout." This is a
hollow shape both for human play and for NN training — agents have no
pressure to engage.

The deeper diagnosis: foedus stripped Diplomacy's negotiation layer
entirely (slated for v3 via Ollama), but kept everything that
*requires* negotiation to be interesting (multi-agent, alliance
dynamics, simultaneous orders). The result is a game whose mechanics
cry out for diplomacy with no diplomacy primitive in place.

This proposal pulls negotiation forward from v3, but as **discrete
structured tokens** — which is what NNs actually train on — with a
**parallel free-text channel** for humans and LLM agents to use
meaningfully. v3's Ollama wrapper becomes a translator on top of this
primitive rather than the primitive itself.

## 2. Goals and non-goals

### Goals
- Make negotiation a first-class engine concept.
- Re-anchor the détente trigger on **active diplomatic consensus**, not
  on passive timeout.
- Add a **stagnation cost** so passive play has an opportunity cost.
- Keep the engine training-friendly (deterministic, fast, no async).
- Preserve full backward compatibility for `RandomAgent`, existing
  tests, and the `play_game()` driver entry point in
  no-press/training mode.
- Give v3's LLM-mediated press a clean substrate to grow into.

### Non-goals
- Free-text NLU. The engine never parses chat; chat is opaque bytes.
- Multi-round haggling within a single turn (deferred; out-of-scope).
- Asymmetric maps, named powers, replay tooling, prettier CLI.
  Tracked separately.
- Binding promises. The engine **observes** intent-vs-action divergence
  but does not enforce intents.

## 3. Locked design decisions

| Dimension | Decision |
|---|---|
| Press content | Stance + non-binding intent + engine-verified commitment |
| Stance visibility | Public to all players |
| Intent visibility | Per-intent recipient set: `frozenset` of recipients OR `None` for public broadcast |
| Betrayal observation visibility | Surfaced to each recipient in the intent's `visible_to` set (or all survivors if public) |
| Cadence | Two-phase turn: NEGOTIATION → ORDERS |
| Negotiation rounds per turn | One |
| Free-text channel | Yes; conversational within the round |
| Token reveal timing | All players' tokens reveal atomically at end of round |
| Token revisability | Revisable until player signals done; locked thereafter |
| Done-signal | One-way (no un-signaling) |
| Round-end condition | All surviving players signaled done **OR** timer expired |
| Default round timer | 60 seconds (configurable; `0` for training mode) |
| Chat message cap | 500 characters (configurable) |
| Chat addressing | Per-message recipient list; `None` = public broadcast |
| Détente re-anchor | Mutual ALLY across all surviving pairs for K turns |
| Default K (`detente_threshold`) | 5 (matches current `peace_threshold` default) |
| Stagnation cost | Flat per-turn score penalty for passive players |
| Default stagnation cost | 1.0 (configurable; `0` to disable) |

## 4. Architecture: engine vs. driver split

The single biggest architectural decision: **the engine never holds a
timer or async event loop**. The engine is a library of pure(-ish) state-
transition functions. Real-time chat orchestration lives in the
*driver* layer (CLI, web UI, training harness).

```
NEGOTIATION phase                         ORDERS phase
─────────────────                         ─────────────
record_chat_message()  ←──┐               finalize_round()
submit_press_tokens()  ←──┤  driver-fed   resolve_turn(orders)
signal_done()          ←──┤               (existing fn, extended)
force_round_end()      ←──┘
```

Same engine serves training (`timer = 0`, no chat, instant
all-done-then-finalize) and live play (real clock, rich chat, players
push done when ready) without forking.

## 5. Data model

All new types live in `foedus/core.py` alongside the existing
frozen-dataclass types.

```python
class Stance(Enum):
    ALLY = "ally"
    NEUTRAL = "neutral"
    HOSTILE = "hostile"

class Phase(Enum):
    NEGOTIATION = "negotiation"
    ORDERS = "orders"

@dataclass(frozen=True)
class Intent:
    """A pre-declaration of what order I will issue for one of my units.

    `visible_to` controls who sees this commitment:
    - `None` → public broadcast: visible to all surviving players
    - `frozenset({p1, p2, ...})` → visible only to the named recipients
    - `frozenset()` → silently dropped on submit (no recipients = no commitment)
    """
    unit_id: UnitId
    declared_order: Order  # reuse existing Order union
    visible_to: frozenset[PlayerId] | None  # None = public broadcast

@dataclass(frozen=True)
class Press:
    """One player's structured outbound press for one round."""
    stance: dict[PlayerId, Stance]               # public; missing = NEUTRAL
    intents: list[Intent]                        # each intent carries its own visibility

@dataclass(frozen=True)
class ChatDraft:
    """What an agent emits when they want to send chat. The engine fills in
    sender, turn, and sequence."""
    recipients: frozenset[PlayerId] | None       # None = public broadcast
    body: str                                    # capped at chat_char_cap

@dataclass(frozen=True)
class ChatMessage:
    """The canonical record stored in chat_history. Constructed by the engine
    from a ChatDraft + the sender + a monotonic round sequence number."""
    turn: int                                    # which turn's round
    sequence: int                                # ordering within the round
    sender: PlayerId
    recipients: frozenset[PlayerId] | None
    body: str

@dataclass(frozen=True)
class BetrayalObservation:
    """End-of-turn signal that someone broke an intent visible to me.

    The intent's `visible_to` set determines who receives this observation:
    public intents broadcast a betrayal observation to all surviving players;
    private/group intents notify only the named recipients.
    """
    turn: int
    betrayer: PlayerId
    intent: Intent
    actual_order: Order
```

### `GameState` additions

```python
@dataclass
class GameState:
    # ... existing fields (turn, map, units, ownership, scores, eliminated,
    #     next_unit_id, config, log) ...

    # Replaces the old peace_streak.
    mutual_ally_streak: int = 0

    # Persistent press archive (one entry per past turn).
    press_history: list[dict[PlayerId, Press]] = field(default_factory=list)
    chat_history: list[list[ChatMessage]] = field(default_factory=list)

    # Per-player observed betrayals (cumulative across the game).
    betrayals: dict[PlayerId, list[BetrayalObservation]] = field(
        default_factory=dict
    )

    # Round-in-progress scratch fields. Only meaningful when phase ==
    # NEGOTIATION; cleared at finalize_round().
    phase: Phase = Phase.NEGOTIATION
    round_chat: list[ChatMessage] = field(default_factory=list)
    round_press_pending: dict[PlayerId, Press] = field(default_factory=dict)
    round_done: set[PlayerId] = field(default_factory=set)
```

### `GameConfig` additions

```python
@dataclass
class GameConfig:
    # ... existing fields ...
    detente_threshold: int = 5          # 0 disables détente entirely
    stagnation_cost: float = 1.0        # 0 to disable
    chat_char_cap: int = 500
    round_timer_seconds: float = 60.0   # default for live play; drivers
                                        # override to 0 for training
                                        # and turn-based CLI play
```

`peace_threshold` is renamed `detente_threshold` for honesty about new
semantics. Keep a deprecation alias for one minor version. Convention
preserved from v1: `detente_threshold = 0` disables the détente
victory condition entirely (single-winner-only games).

## 6. Engine API

New pure functions in `foedus/press.py` (or `foedus/negotiate.py` —
final naming during implementation; not material to the design):

```python
def record_chat_message(state: GameState, sender: PlayerId,
                        draft: ChatDraft) -> GameState:
    """Append a chat message to the current round. Engine constructs the
    canonical ChatMessage from sender + draft + state.turn + monotonic
    sequence number derived from len(state.round_chat).
    Validates: sender alive, all recipients alive, body within cap,
    phase is NEGOTIATION, sender hasn't signaled done. Drops invalid
    silently (returns state unchanged) — driver may surface error."""

def submit_press_tokens(state: GameState, player: PlayerId,
                        press: Press) -> GameState:
    """Set/replace player's pending press tokens. Validates: phase is
    NEGOTIATION, player alive, hasn't signaled done. Multiple calls
    overwrite (revisability)."""

def signal_done(state: GameState, player: PlayerId) -> GameState:
    """Mark player as done negotiating. Idempotent. Cannot be undone."""

def is_round_complete(state: GameState) -> bool:
    """True iff every surviving player has signaled done. Driver also
    checks its own timer; either condition triggers finalize_round."""

def force_round_end(state: GameState) -> GameState:
    """Driver-invoked when timer expires. Marks all not-yet-done
    survivors as done with their most-recent submitted press
    (or empty Press if never submitted)."""

def finalize_round(state: GameState,
                   orders_by_player: dict[PlayerId, dict[UnitId, Order]]
                   ) -> GameState:
    """Atomically: lock press, transition phase to ORDERS, run the full
    resolve_turn logic with the locked press tokens, archive press +
    chat to history, transition back to NEGOTIATION for next turn."""
```

`resolve_turn` is renamed/refactored internally; `finalize_round` is
the public entry point. There is no `start_negotiation` function in
the public API — `initial_state()` returns a GameState in
`Phase.NEGOTIATION` with empty round scratch, and `finalize_round`
returns the next turn's state in the same shape, so the round always
"begins" implicitly.

For backward compatibility, an `advance_turn(state, orders)`
convenience function is provided that internally does
`submit_press_tokens(empty for all) → signal_done(all) →
finalize_round(orders)` — used by `play_game()` in the no-press path
so existing tests keep passing.

## 7. Per-turn state machine

```
Turn N starts  →  phase = NEGOTIATION
                  round_chat = []
                  round_press_pending = {}
                  round_done = set()

[driver loop, until is_round_complete OR timer expired]
   driver → record_chat_message  (any number of times, by any player)
   driver → submit_press_tokens  (per player; revises until done)
   driver → signal_done          (per player; one-way)

[driver detects round complete]
   driver → finalize_round(orders)
       lock press = round_press_pending
       lock chat = round_chat
       run order resolution (see §8)
       run intent verification (see §9)
       update mutual_ally_streak (see §10)
       apply stagnation cost (see §11)
       archive press_history += [press]
       archive chat_history += [chat]
       update betrayals dict per recipient
       phase = NEGOTIATION  (round scratch cleared)
       turn += 1

Turn N+1 starts  →  ...
```

## 8. Order resolution changes

The existing `resolve_turn` logic is intact. The only changes are:

1. It runs as the inner loop of `finalize_round`, not as a public
   entry point.
2. The old `peace_streak` update step is removed; replaced by §10.
3. Two new post-resolution steps: intent verification (§9) and
   stagnation cost (§11).

## 9. Intent verification

For each `(sender, intent)` in the locked round press:

```python
unit = state.units.get(intent.unit_id)
if unit is None or unit.owner != sender:
    continue                       # void; unit dead or not yours

submitted = flat.get(intent.unit_id, Hold())  # raw input, pre-normalize

if submitted != intent.declared_order:
    # Determine which players observe the betrayal.
    if intent.visible_to is None:
        # Public broadcast: every surviving non-sender sees it.
        recipients = {p for p in survivors if p != sender}
    else:
        # Explicit recipient set; filter eliminated and sender.
        recipients = {p for p in intent.visible_to if p in survivors and p != sender}

    for recipient in recipients:
        emit BetrayalObservation(
            turn=state.turn + 1,
            betrayer=sender,
            intent=intent,
            actual_order=submitted,
        ) → recipient
```

**Comparison surface: raw submitted order, not normalized canon, not
resolution outcome.**

- Vs. canon (post-normalization): would falsely flag honest players
  whose intents normalized differently than their orders.
- Vs. outcome (post-resolution): combat luck (bounce, dislodge) is not
  a betrayal signal. Lying is about *what you tried to do*.

## 10. Détente re-anchor

```python
survivors = {p for p in range(num_players) if p not in eliminated}

all_pairs_ally = all(
    locked_press[i].stance.get(j, Stance.NEUTRAL) == Stance.ALLY
    for i in survivors for j in survivors if i != j
)

if all_pairs_ally:
    mutual_ally_streak += 1
else:
    mutual_ally_streak = 0

if mutual_ally_streak >= config.detente_threshold and len(survivors) > 1:
    → terminal; all survivors win (existing winners() logic)
```

If a player is eliminated mid-streak, the survivor matrix shrinks and
the next turn re-evaluates against the new (smaller) survivor set; no
auto-reset. A streak that survives an elimination just means the
remaining players are still all mutually allied.

## 11. Stagnation cost

Applied at end of `finalize_round`, after scoring:

```python
for p in survivors:
    has_units = any(u.owner == p for u in new_units.values())
    if not has_units:
        continue                   # exempt: no units, can't move

    p_canon = [canon[u.id] for u in new_units.values() if u.owner == p]
    if not any(isinstance(o, (Move, SupportMove)) for o in p_canon):
        scores[p] -= config.stagnation_cost
```

**Definition of passive:** all canon orders are `Hold` or `SupportHold`.
A `SupportHold` of any unit (ally or self) counts as passive defense.
A `Move` or `SupportMove` (even one that bounces or fails) counts as
active engagement.

**Default cost (1.0):** exactly cancels the +1 from owning one supply
center this turn. A passive player at supply count 1 nets 0 score gain
versus +1 if they'd been active. At higher supply counts the relative
penalty shrinks, which is intentional — leaders aren't punished for
consolidating, but they don't get a free ride either.

## 12. Fog / visibility rules

`visible_state_for(state, player)` adds the following fields:

| Field | Source | Visibility |
|---|---|---|
| `public_stance_matrix` | `press_history[-1]` (last completed turn's stance) | All survivors see all stance signals |
| `your_outbound_press` | `press_history[-1][player]` | Player sees their own past press in full |
| `your_inbound_intents` | For each sender, the intents in `press_history[-1][sender].intents` whose `visible_to is None` OR `player in visible_to` | Public intents visible to all; private/group intents only to named recipients |
| `your_chat` | Filter `chat_history[-1]` to messages where `recipients is None` OR `player in recipients` OR `sender == player` | Public + DMs to/from this player |
| `your_betrayals` | `state.betrayals.get(player, [])` | Cumulative observations of broken intents to this player |

Note: during a round-in-progress (phase == NEGOTIATION), the fog view
exposes incomplete data — chat as it accumulates, but **not**
round_press_pending of other players (tokens reveal atomically at
finalize, not as submitted). Player's own pending tokens are visible to
themselves.

## 13. Agent protocol changes

```python
class Agent(Protocol):
    def choose_orders(self, state: GameState, player: PlayerId
                      ) -> dict[UnitId, Order]: ...

    # NEW — defaults to empty press for backward-compat.
    def choose_press(self, state: GameState, player: PlayerId) -> Press:
        return Press(stance={}, intents=[])

    # NEW — defaults to no chat. Optional override for live agents.
    # Returns 0 or more chat drafts to send THIS round. Engine fills
    # in sender, turn, and sequence when it constructs the canonical
    # ChatMessage from each draft.
    def chat_drafts(self, state: GameState, player: PlayerId
                    ) -> list[ChatDraft]:
        return []
```

For training, `chat_drafts` is never called (driver in training mode
skips it). For LLM agents, this is the integration point where the LLM
generates chat text and the structured tokens it commits to. Drivers
may call `chat_drafts` once per round (single-shot) or multiple times
within a round (interactive); engine semantics are the same either way.

`RandomAgent` inherits all defaults — emits empty press, no chat.
Existing tests pass without modification.

  - Unit dead before turn: no betrayal regardless of intent/order

  - Same intent sent to multiple recipients, then betrayed: each

## 17. Backward compatibility

- `RandomAgent` works unchanged — emits empty press via Protocol
  defaults.
- `play_game(agents, config)` works unchanged — drives `advance_turn`
  per turn, which emits empty press and instant-completes the round.
- Existing tests pass (with the rename of `peace_threshold` →
  `detente_threshold` aliased; or by leaving the old name as a
  deprecated alias for one minor version).
- The empirical "random-vs-random reaches détente at turn 5"
  observation **disappears** because the détente trigger is no longer
  passive. Random agents will (correctly) play out games to the turn
  cap or until eliminations resolve them.

## 18. What's deferred

- **Multi-round haggling within a turn** (offer → counter → accept as
  separate engine events). Single round per turn is sufficient for v0;
  multi-round is a config knob (`negotiation_rounds: int = 1`) we can
  flip later.
- **Real-time chat orchestration** (websockets, async, etc.). Engine
  exposes pure state-transition primitives; live drivers are
  out-of-scope for this spec.
- **LLM-mediated press translator** (chat ↔ structured tokens). v3.
  Press v0 is the substrate it lands on.
- **Replay tooling** that surfaces the chat + press history visually.
  Future v1.5 work.
- **Heuristic agent** that uses press meaningfully. Future v1.5 work.
- **CLI redesign** beyond minimal text prompts. Tracked separately.
- **Asymmetric maps, named powers, color theming.** Independent
  concerns, not blocked on this.

## 20. Appendix — example turn

Turn 3 of a 4-player game. Players 0, 1, 2, 3 all alive.

**Negotiation phase begins.**

Driver records the following events in order:

1. `record_chat_message(0 → all, "Anyone want to gang up on player 1?")`
2. `record_chat_message(2 → 0, "Maybe. What's in it for me?")`  *(DM)*
3. `record_chat_message(0 → 2, "I'll support your u5 hold at n12.")`  *(DM)*
4. `submit_press_tokens(0, Press(stance={1: HOSTILE, 2: ALLY, 3: NEUTRAL}, intents=[Intent(u3, SupportHold(target=u5), visible_to=frozenset({2}))]))`
5. `submit_press_tokens(2, Press(stance={0: ALLY, 1: HOSTILE, 3: NEUTRAL}, intents=[]))`
6. `signal_done(0)`
7. `signal_done(2)`
8. *(player 1 chats with 3 privately, both signal done)*
9. `is_round_complete()` → True

**Driver collects orders, then:**

```python
finalize_round(state, orders={
    0: {u3: SupportHold(target=u5), u4: Hold()},
    1: ...,
    2: {u5: Hold(), u6: Move(dest=n14)},
    3: ...,
})
```

**At finalize:**
- All press tokens revealed.
- Order resolution runs (existing logic).
- Intent verification: player 0's intent to player 2 was `SupportHold(u5)`
  on `u3`; submitted order for `u3` was `SupportHold(u5)`. **Match —
  no betrayal.** Trust between 0 and 2 holds.
- Stance matrix evaluation: not all-mutual-ALLY (player 0 declared
  HOSTILE toward 1). `mutual_ally_streak = 0`.
- Stagnation cost: applied to any player whose canon was all
  Hold/SupportHold.
- Press archived to `press_history`. Chat archived to `chat_history`.
- `phase` returns to `NEGOTIATION`. Turn 4 begins.
